# /evolve ‚Äî Cycle Rollover (archive done, re-plan from reality)

The third **post-MVP lifecycle** command (Tier 3). `/listen` gathers signals and `/triage` turns them into stories, but both assume a backlog still exists to append to. `/evolve` owns the transition the harness was missing entirely: **the initial backlog is fully delivered ‚Äî now what?**

It is `/create:new-project` run backwards-then-forwards. Instead of *zero ‚Üí plan*, it is *completed cycle ‚Üí archive ‚Üí re-plan from production evidence ‚Üí fresh cycle N+1 backlog*. Like `/create:new-project`, it **orchestrates existing commands** (`/listen`, `/review:retro`, `/create:project-brief` ‚Üí `requirements` ‚Üí `epics`) rather than reimplementing them ‚Äî the new logic is the close-out gate, the history-preserving archive, and seeding the re-plan with *evidence* instead of a blank interview.

> **Origin** (`docs/todo/_design-notes.md`, "Post-MVP lifecycle gap"): *"Dispatch finished 107/107 stories and landed in stall state. The handoff said 'pick next epic' but there was no next epic."* `/evolve` is the command that answers that handoff.

## Telemetry (run first)

This command is user-typed, so it does not fire `PostToolUse:Skill` ‚Äî without this it leaves no `command_invocation` row and fleet analytics under-count human-driven runs. Self-log the invocation before anything else (best-effort ‚Äî if it fails, continue regardless):

```bash
node .claude/core/telemetry-log.js evolve
```

## Design basis

- **Archive, never delete.** The completed cycle's `_backlog.md` + TODO files move to `docs/todo/_archive/cycle-{N}-{stamp}/` via `git mv` (lineage follows in log/blame) **and** stay browsable on disk, accompanied by a self-contained `_cycle-summary.md`. The core promise is "don't lose the history" ‚Äî so deletion is never on the table.
- **Evolve, don't restart.** The re-plan is seeded with *reality* ‚Äî what cycle N actually shipped, what production telemetry is saying (`/listen`), what the retros learned, what got deferred. A blank-slate re-interview would throw away everything the cycle taught. Evidence-first is the difference between `/evolve` and re-running `/create:new-project`.
- **Gate the close-out.** You never archive a half-done cycle. Unchecked stories block the archive unless `--force` is explicit (and `--force` records the incomplete items in the summary, it doesn't hide them).
- **The stack carries forward.** Architecture (`_project-architecture.md`) is **not** regenerated by default ‚Äî a delivery cycle rarely changes the tech stack. Only brief ‚Üí requirements ‚Üí epics re-run. (`--replan-arch` to include it.)
- **Orchestrate, don't reimplement.** Chain the existing planning pipeline, exactly as `/create:new-project` does. Sub-commands commit their own work; `/evolve` commits the archive + summary + evidence digest.

## Variables

SCOPE: $ARGUMENTS

- `--force` ‚Äî archive even with incomplete (`[ ]`/`[!]`) stories. The incomplete set is recorded in `_cycle-summary.md` under "Unfinished, carried forward" and seeded into the next cycle's backlog ‚Äî never silently dropped.
- `--dry-run` ‚Äî report the close-out plan (what would be archived, the evidence digest, the re-plan path) but write **nothing** ‚Äî no archive, no git mv, no sub-command invocation, no commit.
- `--replan-arch` ‚Äî also regenerate `_project-architecture.md` (default: architecture carries forward unchanged).
- A trailing free-text **cycle label** ‚Äî a friendly name folded into the archive dir + summary (e.g. `/evolve "MVP launch"` ‚Üí `cycle-1-260606-mvp-launch/`). Optional.

## Output

- `docs/todo/_archive/cycle-{N}-{stamp}[-{label}]/` ‚Äî the archived `_backlog.md`, `TODO_{Project}.md`, `TODO_epic*.md`, the cycle's `_project-brief.md` + `_project-requirements.md` snapshots, and `_cycle-summary.md` (the self-contained close-out record).
- `docs/todo/_backlog.md` + `docs/TODO_{Project}.md` ‚Äî **fresh**, for cycle N+1 (regenerated by the chained pipeline).
- `docs/_project-brief.md` + `docs/_project-requirements.md` ‚Äî regenerated for cycle N+1 (old copies live in the archive).
- `docs/.output/work/{stamp}/evolve-evidence.md` ‚Äî the synthesized evidence digest that seeded the re-plan.

`{stamp}` is a single `date +%y%m%d-%H%M` run-stamp, computed once at the start of the run and reused verbatim everywhere below (per the run-stamp convention in CLAUDE.md), so a same-day re-run never collides.

## Workflow

### Step 1: Gate ‚Äî is the cycle closeable?

Compute the run-stamp once: `STAMP=$(date +%y%m%d-%H%M)`. Reuse it for every path below.

- Detect the project name (from the `docs/TODO_*.md` filename, else `_project-context.md`, else the repo basename).
- Read `docs/todo/_backlog.md`, `docs/TODO_{Project}.md`, and every `docs/todo/TODO_epic*.md`.
- Tally story checkboxes across all of them: `[x]` done, `[ ]` open, `[~]` deferred, `[!]` blocked.
- **Hard gate:** if any `[ ]` or `[!]` story exists and `--force` is NOT set ‚Üí **STOP**. Print the incomplete stories (file + line) and:

  ```
  Cycle not complete ‚Äî {K} stories still open/blocked. Finish them (/do, /run-todo),
  or re-run with --force to archive anyway (incomplete items carry forward to the next cycle).
  ```

  Per CLAUDE.md gate posture (F3): hitting this gate means *finish the work, then resume* ‚Äî `--force` is an explicit user override, never the recommended path.
- Determine the cycle number **N**: `1 + (count of existing docs/todo/_archive/cycle-* dirs)`. If `_archive/` is absent, this is cycle 1 (the original `/create:new-project` backlog).
- State the resolved cycle number, project name, and completion tally in the report.

### Step 2: Gather re-planning evidence (the "from reality" inputs)

This is what makes it *evolve* and not *restart*. Collect, then synthesize into the evidence digest:

- **2a. Production signals.** Glob `docs/.output/intake/*.md`. If the newest is missing or older than the most recent archive (stale), **run `/listen` first** to get a fresh sweep. Read the newest intake.
- **2b. Lessons.** Read every `docs/.output/reviews/retro-*.md` (epic retrospectives from this cycle). If none exist, note it ‚Äî a cycle with no retros re-plans on thinner evidence.
- **2c. Deferred / waiting work.** Read the triage ledger `docs/.output/triage/_decisions.md` (deferred items whose revisit window has now arrived) and `docs/todo/_feature-ideas.md`. These are candidate cycle N+1 stories.
- **2d. What shipped.** Extract the cycle's delivered scope ‚Äî the `[x]` stories from the about-to-be-archived backlog, plus the headline commits (`git log` since the cycle's first commit / the prior archive date).
- **Synthesize** all of the above into the close-out narrative used in Step 4's summary and Step 5's seed.

### Step 3: Archive the completed cycle (history-preserving) ‚Äî skip all writes if `--dry-run`

- `mkdir -p docs/todo/_archive/cycle-{N}-{STAMP}[-{label-slug}]/`
- Move the cycle's files in with **`git mv`** (preserves history; log/blame follow the move). Fall back to plain `mv` for any untracked file:
  ```bash
  git mv docs/todo/_backlog.md docs/TODO_{Project}.md docs/todo/TODO_epic*.md \
         docs/todo/_archive/cycle-{N}-{STAMP}/
  ```
- **Snapshot the cycle's planning context** into the archive too (copy, not move ‚Äî the create-chain will overwrite the live ones in Step 5):
  ```bash
  cp docs/_project-brief.md docs/_project-requirements.md docs/todo/_archive/cycle-{N}-{STAMP}/
  ```
- **Do NOT archive `_feature-ideas.md`** ‚Äî it is the deferred-ideas bridge between cycles and carries forward into the next backlog.

### Step 4: Write `_cycle-summary.md` (the self-contained close-out record)

Into the archive dir, so the folder alone tells the whole story of cycle N:

```markdown
# Cycle {N} Summary ‚Äî {Project} {label}

**Closed:** {YYYY-MM-DD}  ¬∑  **Span:** {first commit date} ‚Üí {today}  ¬∑  **Stamp:** {STAMP}
**Completion:** {D} done / {O} open / {DF} deferred / {B} blocked  ({pct}%)

## What shipped
- Epic {NN}: {title} ‚Äî {n} stories ({headline commits})
- ...

## What production is telling us  (from /listen intake {date})
- {top signals, by source}

## Lessons  (from {n} retros)
- {key pattern / what worked / what didn't}

## Carried forward to cycle {N+1}
- Deferred (triage ledger / feature-ideas): {items}
- Unfinished, carried forward (only if --force): {open/blocked stories}
```

### Step 5: Regenerate the plan for cycle N+1 ‚Äî skip if `--dry-run`

- Write the evidence digest to `docs/.output/work/{STAMP}/evolve-evidence.md` ‚Äî the Step 2 synthesis formatted as upstream context (vision-relevant signals, validated demand from shipped scope, lessons, carried-forward items).
- Chain the planning pipeline, **each invocation seeded with the evidence digest as context** (exactly how `/create:new-project` passes interview answers down). Because the live planning docs were snapshotted to the archive in Step 3, regeneration is safe:
  1. **`/create:project-brief`** ‚Äî pass `docs/.output/work/{STAMP}/evolve-evidence.md` as the input context (Context Mode); answer "replace" when it detects the existing brief. Produces the cycle N+1 vision, grounded in what shipped + what production wants.
  2. **`/create:project-requirements`** ‚Äî delta PRD from the new brief + evidence.
  3. **`/create:project-architecture`** ‚Äî **skip by default** (stack carries forward). Run only if `--replan-arch`.
  4. **`/create:project-epics`** ‚Äî fresh `docs/todo/_backlog.md` for cycle N+1, seeded with the carried-forward deferred items as candidate stories.
  5. **`/create:project-todo`** ‚Äî regenerate the master `docs/TODO_{Project}.md` index.
- Result: the active `docs/todo/` workspace holds a clean cycle N+1 backlog; cycle N lives entirely in `_archive/`.

### Step 6: Commit (main agent) ‚Äî skip if `--dry-run`

The chained sub-commands (Step 5) commit their own regenerated docs per the Post-Command Commit Convention. **This step commits only what `/evolve` itself produced:** the archive dir (git-mv'd files + snapshots + `_cycle-summary.md`) and the evidence digest.

Follow CLAUDE.md's Post-Command Commit Convention. Stage specifically (never `git add .`):
```bash
git add docs/todo/_archive/cycle-{N}-{STAMP}/ docs/.output/work/{STAMP}/evolve-evidence.md
```
Write the message to `docs/.output/.commit-msg` (Write tool), then `node .claude/core/commit.js` (inline `git commit -m` is blocked):
```
feat: /evolve ‚Äî cycle {N} archived, cycle {N+1} planned

{one-line: what shipped in N, what seeds N+1}
```

### Step 7: Report

```
## /evolve ‚Äî cycle {N} ‚Üí {N+1}

**Project:** {name}  ¬∑  **Cycle {N} completion:** {pct}% ({D}/{total})
**Archive:** docs/todo/_archive/cycle-{N}-{STAMP}/
**Commit:** {hash, or "dry-run ‚Äî nothing written"}

### Cycle {N} closed
- Shipped: {epic/story headline counts}
- Lessons folded in: {n retros, n intake signals}

### Cycle {N+1} planned
- Brief: {1-line new vision}
- Backlog: {epic count} epics, {story count} stories ({n} carried forward from deferred)
- Architecture: {regenerated | carried forward unchanged}

**Next:** `/review:check-readiness` to validate the new cycle, then `/run-todo` or `/do {first story}`.
```

If `--dry-run`, render the same report but mark each write as *(would write)* and end with the literal: `Dry run ‚Äî nothing archived or regenerated.`

## Anti-Patterns

- **Archiving an incomplete cycle silently.** The gate (Step 1) blocks open/blocked stories. `--force` is an explicit override that *records* the incomplete set in the summary and carries it forward ‚Äî it never hides it.
- **Re-planning from a blank slate.** The whole point is evidence-seeded regeneration. If there is genuinely zero evidence (no intake, no retros, no deferred work), say so plainly in the report ‚Äî do **not** fabricate a v2 vision out of nothing.
- **Deleting instead of archiving.** Always `git mv` (or `cp` for snapshots). `rm` of cycle history is never correct ‚Äî "don't lose the history" is the command's reason to exist.
- **Archiving `_feature-ideas.md`.** It bridges cycles and carries forward. Only the backlog + TODO files + planning snapshots get archived.
- **Regenerating architecture by reflex.** The stack carries forward unless `--replan-arch`. Re-running it churns ADRs for no reason on most cycles.
- **Chat-only output.** The archive, summary, evidence digest, and fresh backlog are the artifacts ‚Äî they hit disk before the report (except `--dry-run`, which writes nothing and says so).
